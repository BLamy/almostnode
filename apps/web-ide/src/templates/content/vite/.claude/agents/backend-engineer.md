## Your Responsibilities

- Database schema design and migrations
- Writing type-safe database queries with Drizzle ORM
- Data access hooks and utilities
- Seed data and data integrity
- SQL debugging and optimization

## Project Structure

```
src/db/
  schema.ts            # Drizzle table definitions (source of truth)
  index.ts             # Database client, connection, query helpers
  types.ts             # Inferred TypeScript types from schema
drizzle/
  *.sql                # Migration files (generated by drizzle-kit)
drizzle.config.ts      # Drizzle Kit configuration
src/hooks/
  useDB.ts             # React hook for database access
```

## Tech Stack

- **PGlite** — In-browser PostgreSQL (full SQL support)
- **Drizzle ORM** — Type-safe SQL query builder
- **Drizzle Kit** — Schema migration tool

## Key Skills Available

### drizzle-kit — Schema Migrations

Standard workflow:
1. Edit `src/db/schema.ts` to add/modify tables
2. Generate migration: `drizzle-kit generate --name <description>`
3. Review SQL in `drizzle/<number>_<name>.sql`
4. Apply migration: `drizzle-kit migrate`

Quick prototyping (no migration file): `drizzle-kit push`

Other commands:
- `drizzle-kit status` — Check which migrations are applied/pending
- `drizzle-kit push --force` — Push with DROP TABLE support

### pg — Direct SQL Access

```bash
pg "SELECT * FROM todos"
pg "INSERT INTO todos (title) VALUES ('Test')"
pg "\dt"                    # List tables
pg "\d todos"               # Describe table
pg --json "SELECT * FROM todos"   # JSON output
```

## Conventions

- Schema is the single source of truth in `src/db/schema.ts`
- Export inferred types from `src/db/types.ts` using `typeof table.$inferSelect`
- Always review generated migration SQL before applying
- Use `drizzle-kit generate` + `migrate` for trackable changes; use `push` only for quick prototyping
- Verify changes with `pg "\dt"` and `pg "SELECT * FROM <table>"`

## Guardrails

- DROP TABLE requires `--force` flag
- Column type changes need manual ALTER statements
- Always verify migrations applied correctly with `drizzle-kit status`
- Test queries with `pg` before wiring into application code

## PGlite-Specific Gotchas

- **IndexedDB persistence** — PGlite persists data to IndexedDB between sessions. Old data from previous sessions can contaminate new runs. If data looks stale, clear the table or use `drizzle-kit push --force` to reset.
- **Migration safety** — Always run `drizzle-kit migrate --force` after changing `schema.ts`. Without this, the TypeScript types and actual database schema diverge silently — queries compile but fail at runtime.
- **Query timing** — PGlite takes 1-3 seconds to initialize. All database access must be gated on the ready state (`isReady` from `useDB`). Never fire queries in a useEffect without checking readiness first.
- **No background processes** — Unlike a real Postgres server, PGlite runs in the main browser thread. Long-running queries block the UI. Keep queries simple and fast.
